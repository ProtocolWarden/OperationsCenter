## Overview

Recent fixes to the board-worker promotion pipeline (P3 open-PR gate, thin-goal re-promotion loops, STALE_IN_REVIEW false-positives) have brought the queue to a drained state. This campaign adds focused unit test coverage for promotion decision gates and observability improvements to prevent regression as the system scales.

## Goals

1. **Add P3 open-PR gate unit tests** — Test the P3 open-PR gate in `src/operations_center/board/worker.py` (commit e746942): verify P3 tasks with open PRs are blocked, P3 tasks with closed PRs are eligible, and P0-P2 tasks bypass the gate. Measurable: ≥6 test cases with mocked PlaneClient, each covering one gate path (open PR, closed PR, merged PR, missing PR field, P0–P2 bypass, API error).

2. **Add thin-goal re-promotion regression tests** — Test that the fix in commit bf68fcf prevents thin-goal tasks from being re-promoted inappropriately. Measurable: ≥5 test cases covering task state transitions (enqueue → promote → reset → verify no re-promote), with 2+ distinct thin-goal types (proposal task, bug fix task), and 1+ edge case (task resolved during promotion cycle).

3. **Add STALE_IN_REVIEW false-positive protection tests** — Test the fix in commit a5a088d: verify STALE_IN_REVIEW detection correctly identifies stale tasks with open PRs older than threshold, and does NOT fire when a PR is present (regardless of age). Measurable: ≥4 test cases: open PR (recent commit), open PR (old commit), closed PR (marked stale before closure), PR reopened (not stale after reopening).

4. **Add promotion decision audit logging** — Add structured logging to `src/operations_center/board/unblock.py` to emit `{decision_id, task_id, reason, blocked_by}` for every promotion decision. Write tests that verify logs are emitted and parseable. Measurable: logging added to 2+ decision points, ≥3 test cases verify log structure and grep-ability, CI can extract decision lineage from logs.
