## Overview

Recent commits (8907772d through 8e6b38a1) show active development of model-aware backend selection and per-model worker cooldown gating. The execution layer now distinguishes between per-model rate limits and account-wide limits, and makes backend decisions based on which specific models are available. This campaign hardens the backend selection and rate-limit parsing logic through integration tests, validation of the gating mechanism under load, and end-to-end verification that the cooldown system prevents false-parking on burnt models while allowing available ones.

## Goals

1. **Backend selection under multi-model constraints** — Add integration tests for `worker_backend_selector` and `current_worker_backend_cooldowns()` covering: single-model cooling (sonnet weekly → fall back to opus/haiku), multi-model partial cooling (sonnet burnt, others available), and all-models cooling (account-wide limit). Verify selection respects the distinction between per-model and account-wide limits. Success: 5–7 tests, all constraint combinations validated.

Test edge cases: timezone boundary crossings, missing timezone, multiple models in error log. Success: 10–12 tests, all message formats handled, zero missed limits.

3. **Backend probe and cooldown clearing** — Add tests for `worker_backend_probe.refresh_cooldowns()` to verify stale-cooldown detection and clearing. Cover: successful probe clears single-model cooldown, account-wide probe clears all models, no-op when nothing cooling, timeout handling. Verify probe is safe to run on schedule without side effects. Success: 6–8 tests, all probe outcomes validated, integration with board_unblock gate tested.

4. **End-to-end cooldown and fallback flow** — Add integration tests verifying the complete path from rate-limit response → cooldown recorded → backend selection updated → board unblocks. Test both happy path (probe clears, dispatch resumes) and slow-clear path (fallback model works while cooling). Success: 3–4 tests covering full cycle, realistic response timing.
