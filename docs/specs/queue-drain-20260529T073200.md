## Overview

Hardening and refinement of the queue-drain architecture (board_worker + board_unblock) following recent P3/P6 improvements (thin-goal filtering, STALE_IN_REVIEW PR checks, spec-author payload validation). Four focused improvements to reduce false retries, improve classification accuracy, and prevent redundant task spawning.

## Goals

1. **Spec-author payload validation robustness** — Replace loose yaml.safe_load + try/except with strict Pydantic validation of required fields (spec_slug, target_path, trigger_source, task_phase) and clear error messages. Prevents malformed spec-author tasks from consuming executor quota.

2. **Transient failure classification refinement** — Expand _is_transient_failure reason patterns to cover more network/infrastructure transients (connection timeout variants, gateway errors) and add test coverage for edge cases. Reduces false negatives (legitimate retries treated as permanent failures).

3. **Improve suggestion deduplication** — Implement suggestion fingerprinting (title + rationale hash) in board_worker's improve success handler to prevent duplicate follow-up task spawning when improve_campaign runs produce identical suggestions across multiple attempts.

4. **PR state staleness detection in Rule 5** — Enhance STALE_IN_REVIEW rule (board_unblock) to check actual PR activity (recent comments, reviews, commits) via GitHub API instead of relying solely on task age. Prevents false-positive stale detection for actively-reviewed PRs that haven't merged yet.

## Constraints

- Do not modify board_worker's planning or execution paths — only hardening at claim/success boundaries
- Do not add new Plane API calls in the hot path (claim/dispatch loop) — GitHub checks for STALE_IN_REVIEW are advisory-only
- Keep spec-author validation synchronous (fail fast at claim time, not in execute subprocess)
- Transient pattern additions must be conservative (low false-positive rate; unclear patterns require human review)
- Improve suggestion dedup uses local workspace history only — no Plane re-queries in success handler
