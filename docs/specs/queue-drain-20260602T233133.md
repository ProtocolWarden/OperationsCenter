## Overview

Recent commits (#226, #229) hardened OperationsCenter's PR review automation by making CI-green a merge precondition (not an auto-merge trigger) and closing gaps in the verdict-gate consensus loop. The reviewer watcher now gates merges on both verdict quality and CI status, but the verdict-scoring logic and verdict → merge → dispatch flow lack comprehensive integration test coverage. This campaign adds focused tests for verdict thresholds, CI-green validation, auto-fix loop classification, and edge cases found in recent audits.

## Goals

1. **Verdict-gate consensus logic and threshold tests** — Add unit tests for `VerdictScore.calculate()`, consensus aggregation, and threshold decisions. Cover: unanimous LGTM, mixed votes with majority LGTM, all CONCERNS, and tie-breaking logic. Verify that verdict gates correctly block on score < configured threshold. Success: ≥10 tests, all consensus paths exercised, no silent threshold bypasses.

2. **CI-green merge gate validation** — Add integration tests validating that CI-green is now a mandatory precondition before merge. Cover: PR with passing CI → merge allowed; PR with failing CI → merge blocked; PR with pending CI → merge deferred. Verify gate respects `require_explicit_approval` config. Success: 4–5 tests, CI status correctly blocks/permits merges, no regressions in watcher poll cycle.

3. **Auto-fix loop revision classification** — Add tests for `classify_revision_outcome()` and `should_attempt_revision()` logic. Cover: success (all concerns resolved), failure (same concern persists), partial (some concerns resolved). Verify revision cap is respected and loop terminates correctly. Success: 6–8 tests, all classification branches covered, revision loop bounded.

4. **Verdict-gate edge cases and error recovery** — Add tests for adversarial inputs: corrupted verdict responses, timeout on verdict fetch, missing check runs, malformed CI status payloads. Verify watcher logs clearly, retries safely, and does not corrupt PR state. Success: 5–6 tests covering failure modes, zero silent error swallows, clear diagnostics in logs.

## Constraints

- **Scope:** Tests and validation only — no refactoring of verdict logic, CI polling, or merge decision policies.
- **Allowed paths:** Add tests to `tests/integration/reviewer/` and `tests/unit/reviewer/`; use existing pytest fixtures and mocking patterns.
- **Avoid:** Mocking GitHub's verdict/CI APIs in ways that hide real parsing bugs; use realistic response payloads from recent PR runs when available.
- **Isolation:** All tests must run in parallel and be deterministic. Use fixed timestamps and synthetic PR/verdict fixtures, not live GitHub state.
- **No GitHub calls:** Tests may not make real GitHub API calls; all interactions via mocks or fixtures.
