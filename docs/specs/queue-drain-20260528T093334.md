## Overview

Add comprehensive unit test coverage for board-unblock autonomous queue management rules (Rules 1–9). The board_unblock module is critical to queue health but currently has no dedicated unit tests. Recent rule additions (Rule 9: SPEC_AUTHOR_BACKLOG_PROMOTE) and extensions (Rule 7: improvement_applied tasks) require verification to prevent re-queue gaps and ensure robust state transitions under edge cases.

## Goals

1. **Unit test suite for board-unblock rule evaluation** (Rules 1–8): Create `tests/unit/entrypoints/maintenance/test_board_unblock_rules.py` with parametrized tests covering all eight rule conditions, state transitions, label parsing, and memory threshold behavior. 2. **Rule interaction and precedence tests**: Verify rule application order and mutual exclusivity (e.g., Rule 1 cancels before Rule 4 re-queues; Rule 3 vs Rule 8 Blocked-state handling). Add test cases for overlapping label conditions to ensure correct prioritization.

4. **Integration test for full rule pipeline**: Create `tests/integration/entrypoints/test_board_unblock_e2e.py` with mock PlaneClient to verify end-to-end rule application, state transitions, comment generation, and JSON output for a representative mix of 20+ task scenarios (dead-remediation, blocked-by dependencies, stale states, memory-gated rules).
