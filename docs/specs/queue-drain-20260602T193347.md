## Overview

Harden the CI coordinator decision state machine following the recent change (be1d2b3) that made CI-green a merge precondition instead of an auto-merge trigger. Add comprehensive edge case tests for state transitions, fix-loop logic, and stale PR detection to prevent merge state corruption and ensure safe PR lifecycle handling under concurrent CI events.

## Goals

1. **Test CI-green state transitions under precondition semantics** — Add test cases covering red→defer recovery, green state persistence on re-evaluation, and timeout handling after CI completes. Verify the state machine correctly distinguishes "CI passed" (precondition met) from "auto-merge enabled" (user intent). Acceptance: ≥3 new test cases, all passing; coverage for defer timeout and state re-evaluation paths.

2. **Test self-review and LGTM→merge transitions** — Add cases for self-review promotion (LGTM + CI-green), merge gate validation (all checks pass before merge), and LGTM removal during in-flight CI. Ensure state doesn't advance to merge-ready if CI-green precondition reverts during the transition. Acceptance: ≥3 new test cases; verified merge gate holds on CI regression.

3. **Test fix-loop edge cases and already-passing checks** — Add cases for fix-loop triggering on already-passing checks (should defer, not re-run), concurrent fix attempts, and fix-loop exhaustion (bail after N attempts). Verify the loop respects cooldown and doesn't spam re-evaluations. Acceptance: ≥2 new test cases; fix-loop cooldown enforced in tests.

4. **Test stale PR detection and re-evaluation after state change** — Add cases for detecting stale PRs (no re-evaluation within T window), re-triggering evaluation on CI-green change, and handling force-push during defer. Ensure state machine treats each CI event as a potential state inflection. Acceptance: ≥2 new test cases; stale timeout and force-push re-evaluation verified.
